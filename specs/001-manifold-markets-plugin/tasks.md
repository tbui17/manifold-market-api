## Dependencies & Execution Order

### Phase Dependencies

- **Setup (Phase 1)**: No dependencies — can start immediately
- **Foundational (Phase 2)**: Depends on Phase 1 (directory structure, deps)
- **US1 (Phase 3)**: Depends on Phase 2 (api-client, errors, tool-builder). No dependency on other stories.
- **US2 (Phase 4)**: Depends on Phase 2. Independent of US1 (different tool modules), but `src/index.ts` wiring (T018) builds on US1's wiring (T010).
- **US3 (Phase 5)**: Depends on Phase 2. Independent of US1/US2 tool modules, but `src/index.ts` wiring (T024) builds on prior wiring.
- **US4 (Phase 6)**: Depends on Phases 3-5 (needs all 40 tools defined to expose via MCP server).
- **Polish (Phase 7)**: Depends on all prior phases.

### User Story Dependencies

- **User Story 1 (P1)**: Can start after Foundational (Phase 2). No dependencies on other stories. → **MVP**
- **User Story 2 (P2)**: Can start after Foundational. Tool modules independent of US1; index.ts wiring is sequential after US1.
- **User Story 3 (P3)**: Can start after Foundational. Tool modules independent; index.ts wiring sequential after US2.
- **User Story 4 (P4)**: Depends on all tool modules (US1-US3) being complete — the MCP server exposes the full inventory.

### Within Each User Story

- Tests are written FIRST and MUST FAIL before implementation (TDD per FR-020)
- Tool module creation tasks marked [P] can run in parallel (different files)
- `src/index.ts` wiring and `openclaw.plugin.json` update are sequential (depend on tool modules)
- Validate (`plugin:validate` + `test`) is the final task per story

### Parallel Opportunities

- **Phase 2**: T004, T005, T006, T007 — errors.ts and api-client.ts are sequential (api-client imports errors), but tool-builder.ts (T006) and openclaw.plugin.json (T007) can parallel after T005.
- **Phase 3**: T008 (tests) and T009 (markets.ts) can run in parallel.
- **Phase 4**: T013 (tests), T014 (users.ts), T015 (activity.ts), T016 (groups.ts), T017 (misc-read.ts) — all different files, all parallelizable.
- **Phase 5**: T021 (tests), T022 (authed-read.ts), T023 (write.ts) — all different files, all parallelizable.
- **Phase 6**: T027 (tests) and T028 (mcp-server.ts) can run in parallel.
- **Phase 7**: T030, T031, T032, T033 — all parallelizable.

## Implementation Strategy

### MVP First (User Story 1 Only)

1. Complete Phase 1: Setup (T001-T003)
2. Complete Phase 2: Foundational (T004-T007) — shared client, errors, tool builder
3. Complete Phase 3: US1 (T008-T012) — 7 market read-only tools
4. **MVP delivered**: Users can search markets, fetch by ID/slug, read probabilities, list markets, get positions — all without credentials.

### Incremental Delivery

1. Complete Setup + Foundational → Foundation ready
2. Complete US1 → Market discovery MVP (7 tools, no auth)
3. Complete US2 → Full read-only coverage (20 tools, no auth) — all ~16+ read-only endpoints live-tested
4. Complete US3 → Full authenticated coverage (40 tools total) — trading, market management, personal data
5. Complete US4 → Both deliverables (OpenClaw plugin + standalone MCP server)
6. Complete Polish → All gates pass, publish-ready

### Parallel Team Strategy

With multiple developers:

1. Team completes Setup + Foundational together
2. Developer A: US1 market tools (T008-T012)
3. Developer B: US2 tool modules in parallel (T014-T017), then wires (T018-T020)
4. Developer C: US3 tool modules in parallel (T022-T023), then wires (T024-T026)
5. After US1-US3: Developer D builds MCP server (T027-T029)
6. Team converges for Polish (T030-T033)

**Note**: US2 and US3 tool modules are file-independent and can be developed concurrently with US1 once Foundational is complete. Only `src/index.ts` wiring and `openclaw.plugin.json` updates are sequential across stories.

## Notes

- [P] tasks = different files, no dependencies on incomplete tasks
- Tests hit the LIVE Manifold API (`https://api.manifold.markets/v0`) — no mocks per constitution Principle IV
- Auth-required tests MUST be clearly separated (skip-if-no-key) per FR-020
- All schemas use TypeBox constructors exclusively (constitution Principle II, FR-008)

- 40 tools total: 20 read-only (no auth) + 5 authenticated read + 15 authenticated write
